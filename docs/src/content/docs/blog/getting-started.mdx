Welcome to iSaaSIT! This guide will walk you through everything you need to know to build and deploy your first **CRM SaaS**.

## What is iSaaSIT?

iSaaSIT is a **CRM SaaS starter kit for agencies** who want to build **client CRMs**â€”powerful, branded relationship management platforms where your customers can manage their contacts, deals, pipelines, and tasks.

It provides:

- **CRM foundation** - Ready-to-deploy customer relationship management
- **Contact management** - Store and organize leads and customers
- **Deal pipelines** - Visual pipeline with drag-and-drop stages
- **Task management** - Track activities and follow-ups
- **Multi-tenant architecture** - Each client sees only their own data
- **Built-in authentication** - Enterprise-grade auth with WorkOS AuthKit
- **Role-based access control** - Admin (you), Staff (your team), and Client (your customers)
- **Real-time database** - Convex powers your backend with live queries

## Prerequisites

Before you begin, make sure you have:

- Node.js 20 or later
- A WorkOS account (free tier available)
- A Convex account (free tier available)

## Quick Setup

### 1. Clone the Repository

```bash
git clone https://github.com/your-org/your-repo.git
cd iSaaSIT
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Configure Environment Variables

```bash
cp .env.local.example .env.local
```

Edit `.env.local` with your WorkOS and Convex credentials.

### 4. Start Development

```bash
npm run dev
```

## Understanding the Architecture

iSaaSIT uses a hierarchical data model:

1. **Organization** - Your agency (the CRM owner)
2. **Customers** - Your clients (companies being managed)
3. **Users** - People who access the system

### CRM Features

The starter kit includes:

- **Contacts** - Individual people associated with customers
- **Deals** - Sales opportunities linked to contacts
- **Pipelines** - Visual deal progression through stages
- **Tasks** - Activities and follow-ups for your team

Each level has proper data isolation, ensuring clients only see their own data.
