This guide explains how the iSaaSIT project is organized.

## Root Directory

```
/
├── src/                    # Frontend source code
├── convex/                 # Backend Convex functions
├── docs/                   # Documentation (Starlight)
├── .planning/              # Project planning documents
├── public/                 # Static assets
├── package.json            # Dependencies and scripts
├── vite.config.ts          # Vite configuration
└── convex.json             # Convex deployment config
```

## Frontend (`src/`)

```
src/
├── routes/                 # TanStack Router routes
│   ├── __root.tsx         # Root layout with providers
│   ├── index.tsx          # Home page (public)
│   ├── callback.tsx       # WorkOS OAuth callback
│   ├── _authenticated.tsx # Protected layout
│   └── _authenticated/    # Protected routes
│       └── authenticated.tsx
├── router.tsx             # Router configuration
├── start.ts               # TanStack Start config
├── app.css                # Global styles (Tailwind v4)
├── routeTree.gen.ts       # Auto-generated routes
└── vite-env.d.ts          # Vite environment types
```

### Route Conventions

| Pattern | Description |
|---------|-------------|
| `routes/index.tsx` | Home page at `/` |
| `routes/about.tsx` | Page at `/about` |
| `routes/_layout.tsx` | Layout wrapper (no URL segment) |
| `routes/_layout/page.tsx` | Nested route |
| `routes/(group)/` | Route group (no URL prefix) |

## Backend (`convex/`)

```
convex/
├── schema.ts              # Database schema
├── myFunctions.ts         # Example functions
├── auth.config.ts         # JWT auth configuration
├── _generated/            # Auto-generated types
│   ├── api.d.ts
│   ├── dataModel.d.ts
│   └── server.d.ts
└── README.md              # Convex docs
```

### Function Types

| Type | Purpose | Example |
|------|---------|---------|
| `query` | Read data | `api.users.get` |
| `mutation` | Write data | `api.users.create` |
| `action` | Side effects | `api.email.send` |

## Auto-Generated Files

These files are generated automatically and should **not** be edited:

- `convex/_generated/*` - Generated by `npx convex dev`
- `src/routeTree.gen.ts` - Generated by TanStack Router

## Configuration Files

| File | Purpose |
|------|---------|
| `convex.json` | Convex deployment and AuthKit settings |
| `vite.config.ts` | Vite + TanStack Start + React |
| `tsconfig.json` | TypeScript configuration |
| `postcss.config.mjs` | PostCSS (for Tailwind) |

## Key Patterns

### Component Organization

Components are co-located with routes or in a shared directory:

```
src/
├── components/            # Shared components
│   ├── ui/               # shadcn/ui components
│   └── layout/           # Layout components
└── routes/
    └── dashboard/
        └── components/   # Dashboard-specific components
```

### Import Aliases

| Alias | Maps To |
|-------|---------|
| `@/*` | `./src/*` |
| `~/*` | `./src/*` |

Example:
```tsx
import { Button } from '@/components/ui/button';
```
